# Lesson 128 — Introduction to Design Patterns

> **Object-Oriented Programming (OOP) Handbook**

---

# 1. What are Design Patterns?

Design Patterns are **proven, reusable solutions** to commonly occurring software design problems.

They are **templates**, not finished code.

```text
Problem
   │
Repeated Again
   │
Reusable Solution
   │
Design Pattern
```

---

# 2. Why Use Design Patterns?

Benefits:

- Reusable solutions
- Better code organization
- Easier maintenance
- Reduced coupling
- Improved scalability
- Common vocabulary among developers

---

# 3. Pattern vs Algorithm

| Design Pattern | Algorithm |
|---------------|-----------|
| Solves design problems | Solves computational problems |
| High-level blueprint | Step-by-step procedure |
| Focuses on structure | Focuses on logic |

---

# 4. Categories of Design Patterns

The GoF (Gang of Four) classified patterns into:

```text
Design Patterns
      │
 ├── Creational
 ├── Structural
 └── Behavioral
```

---

# 5. Creational Patterns

Concerned with **object creation**.

Patterns:

- Singleton
- Factory
- Abstract Factory
- Builder
- Prototype

---

# 6. Structural Patterns

Concerned with **class and object composition**.

Patterns:

- Adapter
- Bridge
- Composite
- Decorator
- Facade
- Flyweight
- Proxy

---

# 7. Behavioral Patterns

Concerned with **communication between objects**.

Patterns:

- Strategy
- Observer
- Command
- State
- Template Method
- Visitor
- Chain of Responsibility

---

# 8. Why They Matter

Without patterns:

```text
Complex
Tightly Coupled
Hard to Maintain
```

With patterns:

```text
Flexible
Reusable
Maintainable
Extensible
```

---

# 9. When to Use Patterns

Use patterns when:

- A recurring design problem appears.
- Simpler solutions no longer scale.
- Flexibility and maintainability are important.

Avoid using patterns just to make code look sophisticated.

Because wrapping three lines of code in six design patterns mainly impresses the person who wrote it.

---

# 10. Design Principles Behind Patterns

Patterns are built on principles such as:

- SOLID
- Encapsulation
- Composition over inheritance
- Program to interfaces
- Low coupling
- High cohesion

# 13. Summary

Design patterns provide standardized solutions to recurring software design problems. They promote maintainable, scalable, and loosely coupled systems while giving developers a shared vocabulary for discussing architecture and implementation choices.
