# 2. The Four Pillars

| Pillar | Key Idea |
|--------|----------|
| Encapsulation | Hide data and expose behavior |
| Abstraction | Show what, hide how |
| Inheritance | Reuse through an IS-A relationship |
| Polymorphism | One interface, many implementations |

Remember: use **composition over inheritance** unless inheritance is a natural fit.

# 9. Performance Checklist

✔ Efficient algorithms

✔ Correct data structures

✔ StringBuilder for repeated concatenation

✔ Minimize unnecessary object creation

✔ Profile before optimizing

The fastest code is often the code that avoids unnecessary work entirely. A surprisingly unpopular philosophy in software and meetings alike.

# 10. Interview Strategy

- Admit uncertainty rather than guessing.

# 15. Final Message

Mastering OOP is less about memorizing syntax and more about learning to design software with clear responsibilities, low coupling, high cohesion, and extensibility. Build projects, review code, practice explaining concepts aloud, and solve real design problems. Those habits matter far more than remembering every keyword by heart.
